Brechas Identificadas:
Tras analizar el estado del arte, se detecta que las soluciones actuales fallan en la personalización de reglas operativas para empresas en crecimiento. Las brechas detectadas son:
Falta de Control Algorítmico: Las herramientas genéricas (como Excel) son pasivas. Existe un vacío en la implementación de lógica de programación que bloquee procesos (ej. no permitir la ejecución de una tarea si no existe una aprobación digital previa en la base de datos).
Ausencia de Validación Estricta de Datos: No existen mecanismos programados que obliguen al cumplimiento de checklists técnicos antes de cerrar una orden de trabajo, lo que permite el error humano.
Inexistencia de una Fuente Única de Verdad: La fragmentación de la información en diversos chats impide la persistencia de datos organizada, algo que solo se resuelve mediante el desarrollo de un sistema con una base de datos relacional.
En conclusión: La brecha principal que este proyecto resolverá es la transición de procesos manuales a un flujo de trabajo controlado por software, donde las reglas de negocio de Nexa Servicios estén integradas directamente en el código del sistema.
