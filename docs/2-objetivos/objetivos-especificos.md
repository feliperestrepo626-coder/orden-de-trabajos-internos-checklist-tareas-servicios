Objetivos Específicos: 
1.Levantar los requerimientos funcionales y las reglas de negocio del área operativa para especificar las necesidades técnicas de Nexa Servicios porque es fundamental documentar qué debe hacer el sistema antes de iniciar el desarrollo.

2.Modelar la arquitectura de la base de datos y las interfaces de usuario para estructurar el flujo de las órdenes de trabajo porque se requiere una organización lógica que garantice la trazabilidad y la integridad de la información.

3.Programar el módulo de servicios con validaciones y checklists obligatorios para automatizar la gestión de tareas internas porque se debe eliminar la informalidad y asegurar que ningún proceso se ejecute sin la aprobación digital debida.

Fase de Análisis: 
Levantar los requerimientos funcionales y las reglas de negocio mediante técnicas de recolección de información para definir las necesidades técnicas de los roles de TI, Operaciones y Talento Humano porque es indispensable documentar el comportamiento esperado del sistema antes de iniciar cualquier fase técnica.

Fase de Diseño: 
Modelar la arquitectura de la base de datos relacional y los prototipos de interfaz de usuario para estructurar visual y lógicamente el flujo de las órdenes de trabajo porque se requiere una organización previa que garantice la integridad de los datos y una experiencia de usuario eficiente para el personal de la empresa.

Fase de Construcción: 
Programar el módulo operativo de servicios integrando validaciones de seguridad y checklists obligatorios para automatizar la asignación y ejecución de tareas internas porque se debe eliminar la dependencia de canales informales y asegurar que las reglas de negocio se cumplan mediante código.

Fase de Pruebas: 
Ejecutar pruebas unitarias y de integración sobre los módulos desarrollados para detectar y corregir errores de lógica o de flujo de datos porque se debe garantizar que el software sea confiable, estable y cumpla con los estándares de calidad exigidos antes de su uso real.

Fase de Despliegue: 
Implementar la solución tecnológica en el entorno de producción de la empresa para habilitar el uso oficial del sistema de gestión de órdenes de trabajo porque es necesario realizar la transición del desorden digital actual hacia un ecosistema de trazabilidad controlada.
